# 8. 并发编程模型

现代应用需要高效地处理并发任务：网络请求、文件 I/O、用户交互等。"道"提供两层并发模型：**异步/等待**用于 I/O 密集型任务，**协程/通道**用于计算密集型并行任务。

## 8.1 异步/等待 (Async/Await)

对于网络请求、文件读写等 I/O 操作，"道"采用 `异步` 和 `等待` 关键字，让异步代码看起来像同步代码一样直观。

### 8.1.1 基础用法

```
// 定义异步函数
异步 函数 获取用户信息(用户ID)
    定义 响应 = 等待 网络.请求(`/api/用户/{用户ID}`)
    返回 响应.数据

// 调用异步函数
异步 函数 主流程()
    定义 用户 = 等待 获取用户信息(123)
    打印(`用户名: {用户.名字}`)
```

### 8.1.2 并行等待

当多个异步操作之间互不依赖时，可以并行执行以节省时间：

```
异步 函数 加载页面数据()
    // 顺序执行（慢）
    // 定义 用户 = 等待 获取用户信息(123)
    // 定义 文章 = 等待 获取文章列表()
    // 定义 评论 = 等待 获取评论列表()

    // 并行执行（快！三个请求同时发出）
    定义 (用户, 文章, 评论) = 等待 全部([
        获取用户信息(123),
        获取文章列表(),
        获取评论列表()
    ])

    返回 {"用户": 用户, "文章": 文章, "评论": 评论}
```

### 8.1.3 竞速等待

等待多个操作中最先完成的那个：

```
异步 函数 获取数据_带超时()
    定义 结果 = 等待 竞速([
        获取远程数据(),
        延迟(5000, 函数() => 抛出 错误("请求超时"))
    ])
    返回 结果
```

### 8.1.4 异步错误处理

异步代码使用与同步代码相同的 `尝试`-`捕获` 结构：

```
异步 函数 安全获取数据()
    尝试
        定义 数据 = 等待 获取远程数据()
        返回 数据
    捕获 错误
        打印(`获取数据失败: {错误.信息}`)
        返回 空
```

## 8.2 协程与通道 (Coroutine & Channel)

对于需要真正并行计算的场景，"道"提供受 Go 语言启发的协程和通道模型。协程是轻量级的并发执行单元，通道用于协程之间安全地传递数据。

### 8.2.1 启动协程

使用 `并行` 关键字启动一个协程：

```
// 启动一个协程
并行
    打印("这在一个独立的协程中运行")
    定义 结果 = 耗时计算()
    打印(`计算完成: {结果}`)

打印("主程序继续执行，不会被阻塞")
```

### 8.2.2 通道通信

通道是协程之间安全通信的管道。使用 `通道` 创建，`发送` 和 `接收` 进行数据传递。

```
// 创建一个通道
定义 数据通道 = 通道()

// 生产者协程
并行
    遍历 i 从 1 到 5
        数据通道.发送(i * 10)
        打印(`已发送: {i * 10}`)
    数据通道.关闭()  // 发送完毕后关闭通道

// 消费者协程
并行
    遍历 数据 在 数据通道  // 遍历通道直到关闭
        打印(`已接收: {数据}`)

// 输出（交替出现）：
// 已发送: 10
// 已接收: 10
// 已发送: 20
// 已接收: 20
// ...
```

### 8.2.3 带缓冲的通道

```
// 创建带缓冲的通道（最多存放3个元素）
定义 缓冲通道 = 通道(容量=3)

// 可以在不阻塞的情况下发送3个值
缓冲通道.发送(1)
缓冲通道.发送(2)
缓冲通道.发送(3)
// 缓冲通道.发送(4)  // 这里会阻塞，直到有消费者取走一个
```

### 8.2.4 选择器 (Select)

从多个通道中选择第一个就绪的：

```
定义 通道甲 = 通道()
定义 通道乙 = 通道()

选择
    当 数据 来自 通道甲
        打印(`从甲收到: {数据}`)
    当 数据 来自 通道乙
        打印(`从乙收到: {数据}`)
    超时 5000
        打印("等待超时")
```

## 8.3 并发安全

### 8.3.1 锁机制

当必须共享可变状态时，使用 `互斥锁`：

```
定义 计数器 = 0
定义 锁 = 互斥锁()

函数 安全递增()
    锁.获取()
    尝试
        计数器 = 计数器 + 1
    最终
        锁.释放()

// 更简洁的写法：使用 `同步` 块
函数 安全递增_v2()
    同步 锁
        计数器 = 计数器 + 1
```

### 8.3.2 原子操作

对于简单的数值操作，使用原子类型避免锁的开销：

```
定义 计数器 = 原子数值(0)

并行
    遍历 _ 从 1 到 1000
        计数器.递增()

并行
    遍历 _ 从 1 到 1000
        计数器.递增()

// 等待所有协程完成后
打印(计数器.值())  // 2000
```

## 8.4 实用示例：并发网络爬虫

```
异步 函数 爬取网页(网址)
    定义 响应 = 等待 网络.请求(网址)
    返回 {"网址": 网址, "内容": 响应.文本}

异步 函数 批量爬取(网址列表, 最大并发数=5)
    定义 结果通道 = 通道()
    定义 信号量 = 通道(容量=最大并发数)

    // 为每个网址启动协程
    遍历 网址 在 网址列表
        信号量.发送(真)  // 占用一个信号量
        并行
            尝试
                定义 结果 = 等待 爬取网页(网址)
                结果通道.发送(结果)
            捕获 错误
                结果通道.发送({"网址": 网址, "错误": 错误.信息})
            最终
                信号量.接收()  // 释放信号量

    // 收集所有结果
    定义 所有结果 = []
    遍历 _ 从 1 到 长度(网址列表)
        所有结果.添加(结果通道.接收())

    返回 所有结果

// 使用
异步 函数 主程序()
    定义 网址列表 = [
        "https://example.com/page1",
        "https://example.com/page2",
        "https://example.com/page3",
    ]

    定义 结果 = 等待 批量爬取(网址列表)
    遍历 项 在 结果
        打印(`{项.网址}: {长度(项.内容)} 字节`)
```
