# 6. 元编程/宏系统 (Metaprogramming/Macro System)

元编程是"代码写代码"的艺术。一个强大的宏系统能让开发者从根本上扩展语言的语法，创建领域特定语言（DSL），并消除重复的模板代码。"道"的宏系统以 Lisp 的同象性（Homoiconicity）思想为指导，并借鉴 Rust 宏的健壮性。

## 6.1 设计理念：受控的同象性

纯粹的同象性，即代码和数据具有完全相同的结构（如 Lisp 的 S 表达式），对于一种具有更传统语法的语言来说难以实现。我们将采用一种"受控的同象性"方法：

1. **代码即数据结构**：提供一种机制，可以将任意"道"代码块转换为语言内置的数据结构，即抽象语法树（AST）的表示。这个 AST 将由列表、字典和基本类型构成。
2. **宏在 AST 上操作**：宏本质上是特殊的函数，它们在编译期执行，接受 AST 作为输入，并返回一个新的 AST。
3. **语法安全**：宏系统应是卫生的（Hygienic Macros），以避免宏引入的变量与使用处的变量发生意外冲突。

## 6.2 核心语法

| 关键字 | 说明 |
|--------|------|
| `定义宏` | 定义一个编译期宏 |
| `引述` | 将代码块转换为 AST 数据（类似 Lisp 的 `quote`） |
| `注入` | 在引述块中插入变量的 AST 值（类似 Lisp 的 `unquote`） |
| `!宏名` | 宏调用前缀，区分宏与普通函数 |

## 6.3 基础示例：除非宏

```
// 设计一个 `除非` 宏，作为 `如果` 的反义词
定义宏 除非(条件, 代码块)
    返回 引述
        如果 不是 (注入 条件)
            注入 代码块

// 使用宏
定义 温度 = 15
!除非(温度 > 20, {
    打印("天气有点冷，请加衣服。")
})

// 上述宏在编译期会被展开为：
// 如果 不是 (温度 > 20)
//     打印("天气有点冷，请加衣服。")
```

## 6.4 实用宏示例

### 6.4.1 计时宏

自动为任意代码块添加计时功能：

```
定义宏 计时(代码块)
    返回 引述
        定义 __开始 = 系统.当前时间()
        定义 __结果 = { 注入 代码块 }
        定义 __耗时 = 系统.当前时间() - __开始
        打印(`执行耗时: {__耗时} 毫秒`)
        __结果

// 使用
!计时({
    定义 总和 = 折叠(范围(1, 1000000), 0, 函数(a, b) => a + b)
    打印(`总和: {总和}`)
})
// 输出：
// 总和: 499999500000
// 执行耗时: 42 毫秒
```

### 6.4.2 重试宏

自动为可能失败的操作添加重试逻辑：

```
定义宏 重试(次数, 代码块)
    返回 引述
        定义 __剩余次数 = 注入 次数
        定义 __最后错误 = 空
        当 __剩余次数 > 0
            尝试
                { 注入 代码块 }
                跳出  // 成功则退出循环
            捕获 错误
                __最后错误 = 错误
                __剩余次数 = __剩余次数 - 1
                如果 __剩余次数 > 0
                    打印(`操作失败，剩余重试次数: {__剩余次数}`)
        如果 __最后错误 != 空 并且 __剩余次数 == 0
            抛出 __最后错误

// 使用：自动重试3次网络请求
!重试(3, {
    定义 响应 = 网络.请求("https://api.example.com/data")
    打印(响应.数据)
})
```

### 6.4.3 日志宏

自动记录函数的输入输出：

```
定义宏 记录日志(函数定义)
    // 解析函数定义的AST，提取函数名和参数
    定义 函数名 = 函数定义.名称
    定义 参数列表 = 函数定义.参数
    定义 函数体 = 函数定义.主体

    返回 引述
        函数 (注入 函数名)((注入 参数列表))
            打印(`[日志] 调用 {(注入 函数名)} 参数: {(注入 参数列表)}`)
            定义 __返回值 = { 注入 函数体 }
            打印(`[日志] 返回: {__返回值}`)
            返回 __返回值

// 使用
!记录日志(
    函数 计算价格(单价, 数量)
        返回 单价 * 数量
)

计算价格(100, 5)
// 输出：
// [日志] 调用 计算价格 参数: [100, 5]
// [日志] 返回: 500
```

### 6.4.4 构建器宏

为领域特定语言（DSL）提供便捷语法：

```
定义宏 网页(代码块)
    // 宏将DSL代码转换为HTML字符串
    返回 解析网页DSL(代码块)

// 使用DSL语法生成HTML
定义 页面 = !网页({
    标题 "我的主页"
    段落 "欢迎来到道语言的世界"
    列表
        项 "简洁"
        项 "强大"
        项 "中文原生"
})

// 展开后等效于：
// 定义 页面 = "<h1>我的主页</h1><p>欢迎来到道语言的世界</p><ul><li>简洁</li><li>强大</li><li>中文原生</li></ul>"
```

## 6.5 AST 内省

开发者可以在运行时检查代码的 AST 结构，用于调试和高级元编程：

```
定义 代码树 = 引述
    函数 你好(名字)
        打印(`你好, {名字}`)

// 检查AST结构
打印(代码树.类型)     // "函数定义"
打印(代码树.名称)     // "你好"
打印(代码树.参数)     // ["名字"]
打印(代码树.子节点)   // [打印调用节点...]
```

## 6.6 宏的卫生性

"道"的宏系统默认是卫生的，宏内部定义的变量不会与调用处的变量冲突：

```
定义宏 交换(甲, 乙)
    返回 引述
        定义 临时 = 注入 甲     // 这里的 `临时` 不会与调用者的同名变量冲突
        注入 甲 = 注入 乙
        注入 乙 = 临时

定义 临时 = "不会被影响"
定义 x = 1
定义 y = 2
!交换(x, y)
打印(x)   // 2
打印(y)   // 1
打印(临时) // "不会被影响" （宏内部的 `临时` 与此处的 `临时` 是不同的变量）
```
