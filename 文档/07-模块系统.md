# 7. 模块系统

模块系统是构建大型应用的基石。它解决了命名冲突、代码组织和依赖管理三大核心问题。"道"的模块系统设计力求简洁直观。

## 7.1 基本概念

- **文件即模块**：每个 `.道` 源文件自动构成一个模块
- **导出**：使用 `导出` 关键字将函数、类型、常量等暴露给外部
- **导入**：使用 `导入` 关键字引入其他模块的功能

## 7.2 导出

### 7.2.1 逐项导出

在需要导出的定义前添加 `导出` 关键字：

```
// 文件：数学工具.道

// 导出函数
导出 函数 加(甲, 乙)
    返回 甲 + 乙

导出 函数 减(甲, 乙)
    返回 甲 - 乙

// 导出常量
导出 常量 圆周率 = 3.14159

// 导出类型
导出 类型 向量
    初始化(x, y)
        本对象.x = x
        本对象.y = y

// 未导出的（模块私有）
函数 内部辅助函数()
    // 外部无法访问此函数
    返回 42
```

### 7.2.2 集中导出

在文件末尾统一导出，便于一目了然地了解模块的公共 API：

```
// 文件：字符串工具.道

函数 大写(文本)
    // ...

函数 小写(文本)
    // ...

函数 修剪(文本)
    // ...

函数 内部处理(文本)
    // 不导出

// 文件末尾集中导出
导出 大写, 小写, 修剪

// 或
// 导出
//     大写,
//     小写,
//     修剪

// 或
// 导出
//     大写, 小写, 修剪
```

## 7.3 导入

### 7.3.1 导入整个模块

```
// 导入整个模块，通过模块名访问
导入 数学工具

打印(数学工具.加(1, 2))     // 3
打印(数学工具.圆周率)       // 3.14159
```

### 7.3.2 导入特定项

```
// 只导入需要的函数
从 数学工具 导入 加, 圆周率

// 或
// 从 数学工具 导入
//     加, 圆周率

// 或
// 从 数学工具 导入
//     加,
//     圆周率


打印(加(1, 2))     // 3
打印(圆周率)       // 3.14159
```

### 7.3.3 重命名导入

```
// 避免命名冲突
从 数学工具 导入 加 作为 数学加法
从 字符串工具 导入 连接 作为 字符串拼接

打印(数学加法(1, 2))
打印(字符串拼接("你好", "世界"))
```

### 7.3.4 导入子目录模块

```
// 目录结构：
// 项目/
// ├── 主程序.道
// ├── 工具/
// │   ├── 数学.道
// │   ├── 文本.道
// │   └── 网络/
// │       └── 请求.道

// 用路径分隔符导入
导入 工具.数学
从 工具.文本 导入 大写
从 工具.网络.请求 导入 发送请求
```

## 7.4 包管理

### 7.4.1 包定义文件

每个项目根目录下有一个 `道.配置` 文件（类似 `project.yml` 配置文件）：

```
// 道.配置

名称 我的项目
版本 1.0.0
作者 张三
描述 一个示例项目
入口 主程序.道

依赖
    网络框架 >=2.0.0
    数据库驱动 ^1.5.0
    日志工具 1.2.3

开发依赖
    测试框架 >=3.0.0
```

### 7.4.2 导入第三方包

```
// 导入第三方包
从 网络框架 导入 创建服务器, 路由
从 数据库驱动 导入 连接数据库
```

## 7.5 模块初始化

模块在首次被导入时执行其顶层代码（仅一次），后续导入直接使用缓存结果：

```
// 文件：数据库.道

// 这段代码在模块首次被导入时执行一次
打印("正在初始化数据库连接...")
定义 数据库连接 = 连接数据库("localhost", 3306)

导出 函数 查询(sql语句)
    返回 数据库连接.执行(sql语句)
```

## 7.6 条件导入

根据运行时环境选择不同的实现：

```
如果 系统.平台 == "浏览器"
    从 网络.浏览器端 导入 发送请求
否则
    从 网络.服务器端 导入 发送请求
```
