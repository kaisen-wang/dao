# 4. 函数式编程 (Functional Programming)

函数式编程（FP）强调使用纯函数、避免共享状态和可变数据。它能使代码更易于推理和测试。"道"将深度集成 FP 的核心特性。

## 4.1 一等公民函数 (First-Class Functions)

函数可以被赋值给变量、作为参数传递和作为返回值。这是函数式编程的基石。

```
// 函数赋值给变量
定义 操作 = 函数(x, y) => x + y
打印(操作(3, 4))  // 7

// 函数作为返回值（闭包）
函数 创建乘法器(倍数)
    返回 函数(x) => x * 倍数

定义 双倍 = 创建乘法器(2)
定义 三倍 = 创建乘法器(3)
打印(双倍(5))  // 10
打印(三倍(5))  // 15
```

## 4.2 匿名函数（Lambda）

"道"提供两种匿名函数语法：

```
// 单表达式简写（箭头语法）

// 无参数，直接箭头
函数 => 表达式

定义 无入参函数 = 函数 => 2 * 2

// 有参数列表
函数(参数) => 表达式

定义 平方 = 函数(x) => x * x

// 多语句写法（代码块语法）
定义 处理 = 函数(x)
    定义 结果 = x * x
    如果 结果 > 100
        返回 100
    返回 结果
```

## 4.3 高阶函数 (Higher-Order Functions)

接受函数作为参数或返回函数的函数。"道"的核心库将提供常见的高阶函数：

| 函数名 | 英文对应 | 说明 |
|--------|---------|------|
| `映射` | map | 对集合中每个元素应用函数 |
| `筛选` | filter | 保留满足条件的元素 |
| `折叠` | reduce/fold | 将集合归约为单个值 |
| `展平映射` | flatMap | 映射后展平嵌套 |
| `每个满足` | every | 是否所有元素都满足条件 |
| `存在满足` | some | 是否存在元素满足条件 |
| `查找` | find | 查找第一个满足条件的元素 |
| `分组` | groupBy | 按条件分组 |

```
定义 数字列表 = [1, 2, 3, 4, 5]

// 映射
定义 平方列表 = 映射(数字列表, 函数(x) => x * x)
// => [1, 4, 9, 16, 25]

// 筛选
定义 偶数列表 = 筛选(平方列表, 函数(x) => x % 2 == 0)
// => [4, 16]

// 折叠
定义 总和 = 折叠(偶数列表, 0, 函数(累加值, 当前值) => 累加值 + 当前值)
// => 20

// 展平映射
定义 嵌套 = [[1, 2], [3, 4], [5]]
定义 展平 = 展平映射(嵌套, 函数(列表) => 映射(列表, 函数(x) => x * 2))
// => [2, 4, 6, 8, 10]

// 分组
定义 人员 = [
    {"姓名": "张三", "部门": "技术"},
    {"姓名": "李四", "部门": "市场"},
    {"姓名": "王五", "部门": "技术"},
]
定义 按部门分组 = 分组(人员, 函数(p) => p.部门)
// => {"技术": [张三, 王五], "市场": [李四]}
```

## 4.4 管道运算符 (Pipe Operator)

管道运算符 `|>` 使数据处理链条更加清晰易读，数据从左向右流动。这避免了深层嵌套的函数调用。

```
// 没有管道运算符时（嵌套调用，从内往外读）：
定义 结果 = 排序(筛选(映射(数据, 转换函数), 条件函数))

// 使用管道运算符（从左往右读，像流水线一样清晰）：
定义 结果 = 数据
    |> 映射(转换函数)
    |> 筛选(条件函数)
    |> 排序()
```

**完整示例：**

```
定义 订单列表 = [
    {"商品": "手机", "价格": 5000, "数量": 1},
    {"商品": "耳机", "价格": 200, "数量": 3},
    {"商品": "笔记本", "价格": 8000, "数量": 1},
    {"商品": "鼠标", "价格": 100, "数量": 2},
]

// 用管道运算符处理数据：
// 1. 计算每单总价 → 2. 筛选大于500的 → 3. 按总价排序 → 4. 提取商品名
定义 结果 = 订单列表
    |> 映射(函数(单) => {
        ...单,
        "总价": 单.价格 * 单.数量
    })
    |> 筛选(函数(单) => 单.总价 > 500)
    |> 排序(键=函数(单) => 单.总价, 降序=真)
    |> 映射(函数(单) => `{单.商品}: {单.总价}元`)

// 结果: ["笔记本: 8000元", "手机: 5000元", "耳机: 600元"]
```

**管道中的匿名函数简写：**

当管道中的函数只有一个参数时，可使用 `_` 占位符进一步简化：

```
定义 结果 = [1, 2, 3, 4, 5]
    |> 筛选(_ > 2)        // 等同于 筛选(函数(x) => x > 2)
    |> 映射(_ * 10)       // 等同于 映射(函数(x) => x * 10)
    // 结果: [30, 40, 50]
```

## 4.5 柯里化 (Currying)

柯里化将多参数函数转换为一系列单参数函数。"道"通过 `柯里化` 内置函数支持自动柯里化。

```
函数 加(甲, 乙)
    返回 甲 + 乙

// 手动部分应用
定义 加五 = 函数(x) => 加(5, x)
打印(加五(3))  // 8

// 使用柯里化工具
定义 柯里加 = 柯里化(加)
定义 加十 = 柯里加(10)
打印(加十(3))   // 13
打印(柯里加(1)(2))  // 3

// 柯里化与管道配合
函数 乘(甲, 乙)
    返回 甲 * 乙

定义 柯里乘 = 柯里化(乘)

定义 结果 = [1, 2, 3, 4, 5]
    |> 映射(柯里乘(10))
    // 结果: [10, 20, 30, 40, 50]
```

## 4.6 函数组合 (Function Composition)

使用 `组合` 将多个函数串联成一个新函数，数据依次经过每个函数处理。

```
函数 加一(x)
    返回 x + 1

函数 双倍(x)
    返回 x * 2

函数 转文本(x)
    返回 `结果是: {x}`

// 组合：从右到左执行（数学传统）
定义 处理 = 组合(转文本, 双倍, 加一)
打印(处理(3))  // "结果是: 8"  (3 → 加一→4 → 双倍→8 → 转文本)

// 管道组合：从左到右执行（更直观）
定义 处理2 = 管道组合(加一, 双倍, 转文本)
打印(处理2(3))  // "结果是: 8"
```

## 4.7 模式匹配 (Pattern Matching)

提供比传统 `如果-否则` 更强大、更具表现力的分支结构。`匹配` 表达式可以解构复杂的数据结构。

### 4.7.1 基础匹配

```
函数 计算斐波那契(n)
    匹配 n
        情况 0: 返回 0
        情况 1: 返回 1
        情况 其他值: 返回 计算斐波那契(其他值 - 1) + 计算斐波那契(其他值 - 2)
```

### 4.7.2 解构匹配

```
函数 处理数据(数据)
    匹配 数据
        情况 []: 打印("空列表")
        情况 [唯一元素]: 打印(`只有一个元素: {唯一元素}`)
        情况 [头, ...尾]: 打印(`头部是 {头}，剩余 {长度(尾)} 个元素`)
        情况 {"姓名": 名字, "年龄": 年龄}: 打印(`{名字} 今年 {年龄} 岁`)
        情况 _: 打印("未知数据结构")  // _ 是通配符
```

### 4.7.3 守卫条件 (Guard)

在模式匹配中添加额外的条件判断。

```
函数 评价分数(分数)
    匹配 分数
        情况 n 当 n >= 90: 返回 "优秀"
        情况 n 当 n >= 80: 返回 "良好"
        情况 n 当 n >= 60: 返回 "及格"
        情况 n 当 n >= 0:  返回 "不及格"
        情况 _: 返回 "无效分数"

// 配合解构使用
函数 处理响应(响应)
    匹配 响应
        情况 {"状态码": 200, "数据": 数据}: 
            返回 数据
        情况 {"状态码": 码} 当 码 >= 400 并且 码 < 500:
            抛出 错误(`客户端错误: {码}`)
        情况 {"状态码": 码} 当 码 >= 500:
            抛出 错误(`服务器错误: {码}`)
```

### 4.7.4 枚举匹配

```
枚举 结果
    成功(值)
    失败(错误信息)

函数 处理结果(r)
    匹配 r
        情况 结果.成功(值):
            打印(`操作成功: {值}`)
        情况 结果.失败(信息):
            打印(`操作失败: {信息}`)
```

## 4.8 惰性求值 (Lazy Evaluation)

对于处理大型或无限数据集，惰性求值是关键。它延迟表达式的计算直到其结果被真正需要时。"道"通过 `流`（Stream/Generator）的概念和 `产出` 关键字来支持惰性求值。

```
// 定义一个无限的自然数流
函数 无限自然数()
    定义 n = 0
    当 真
        产出 n
        n = n + 1

// 流本身不会立即计算所有值
定义 自然数流 = 无限自然数()

// 对流进行操作，这些操作也是惰性的
定义 平方流 = 映射流(自然数流, 函数(x) => x * x)
定义 偶数平方流 = 筛选流(平方流, 函数(x) => x % 2 == 0)

// 只有在需要时才取出结果
定义 前五个结果 = 取(偶数平方流, 5) // => [0, 4, 16, 36, 64]
```

**实用示例：逐行读取大文件**

```
函数 读取行(文件路径)
    定义 文件 = 打开文件(文件路径)
    尝试
        当 不是 文件.已结束()
            产出 文件.读取一行()
    最终
        文件.关闭()

// 处理一个几GB的日志文件，内存只需保存一行
定义 错误行 = 读取行("巨大的日志.txt")
    |> 筛选流(函数(行) => "错误" 在 行)
    |> 取(10)  // 只取前10条错误
```

## 4.9 不可变数据操作

为了支持函数式编程的不可变性原则，"道"提供方便的不可变更新语法：

```
// 展开运算符 ... 用于创建修改后的副本
定义 原始 = {"姓名": "张三", "年龄": 25, "城市": "北京"}

// 创建一个新字典，修改部分字段（原始不变）
定义 更新后 = {...原始, "年龄": 26, "城市": "上海"}
打印(原始.年龄)    // 25（原始未改变）
打印(更新后.年龄)  // 26

// 列表的不可变更新
定义 原列表 = [1, 2, 3]
定义 新列表 = [...原列表, 4, 5]  // [1, 2, 3, 4, 5]
```
