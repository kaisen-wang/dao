# 2. 语言核心设计："道"的语法与关键字

"道"的语法设计旨在实现简洁性、可读性和表现力之间的平衡。它从易语言等现有中文项目中汲取灵感，即使用中文关键字降低理解难度，但同时借鉴 Python 等现代语言的简洁风格，避免冗长和模板化的代码。

## 2.1 设计哲学：表意清晰，消除歧义

中文作为一种自然语言，其词汇和语法具有高度的灵活性和一定的模糊性。在设计编程语言时，首要任务是选择一组核心关键字，确保它们在编程上下文中具有唯一、精确的含义。

* **关键字选择**：关键字应选择现代汉语中常用且含义明确的词语。例如，使用 `定义` 表示变量或函数声明，`如果`、`否则如果`、`否则` 表示条件分支，`当`/`遍历` 表示循环结构。
* **语法结构**：采用"主谓宾"或"状语-主谓"等符合中文语序的结构。例如，函数调用 `打印("你好")` 比 `("你好") 打印` 更自然。
* **标点符号**：合理使用中文和西文标点。例如，使用中文括号 `（` 和 `）` 或西文括号 `(` 和 `)` 包围参数列表（编译器同时支持两种），使用逗号 `,` 分隔参数，使用换行和缩进表示语句块的层级。

## 2.2 注释语法

注释是代码可读性的基石。"道"提供三种注释方式：

### 2.2.1 单行注释

使用 `//` 或中文符号 `注：` 开头，到行尾结束。

```
// 这是一条单行注释
注：这也是一条单行注释，更贴近中文阅读习惯

定义 名字 = "张三" // 行尾注释
```

### 2.2.2 多行注释

使用 `/*` 和 `*/` 包围，支持嵌套。

```
/*
  这是一段多行注释。
  可以跨越多行。
  /* 支持嵌套注释，方便临时注释掉含注释的代码段 */
*/
```

### 2.2.3 文档注释

使用 `///` 开头的连续行，用于为函数、类型等生成 API 文档。

```
/// 计算两个数的和
/// 参数 甲：第一个加数
/// 参数 乙：第二个加数
/// 返回：两个数之和
函数 加法(甲, 乙)
    返回 甲 + 乙
```

## 2.3 基础语法结构

### 2.3.1 变量与常量定义

使用 `定义` 关键字声明一个变量。为了支持函数式编程中的不可变性，我们引入 `常量` 关键字。

```
// 定义一个可变变量
定义 名字 = "张三"
名字 = "李四" // 合法

// 定义一个不可变常量
常量 圆周率 = 3.14159
圆周率 = 3.14 // 这将导致编译错误或运行时错误
```

### 2.3.2 数据类型

作为动态类型语言，"道"在定义变量时无需声明类型。其内置基础数据类型包括：

| 类型名 | 说明 | 字面量示例 |
|--------|------|-----------|
| **数值** | 整数和浮点数，支持下划线分隔 | `42`, `3.14`, `1_000_000` |
| **文本** | 字符串，双引号或单引号 | `"你好"`, `'世界'` |
| **布尔** | 逻辑真假 | `真`, `假` |
| **空值** | 表示"无" | `空` |
| **列表** | 有序可变集合 | `[1, "苹果", 真]` |
| **元组** | 有序不可变集合 | `(1, "苹果", 真)` |
| **字典** | 键值对集合 | `{"姓名": "王五", "年龄": 30}` |
| **集合** | 无序不重复集合 | `集合(1, 2, 3)` 或 `{1, 2, 3}` |
| **枚举** | 命名的常量集 | 见下方示例 |

**元组 (Tuple)**：一旦创建就不可修改，适用于函数返回多个值的场景。

```
定义 坐标 = (10, 20)
定义 甲 = 坐标.0  // 10，通过索引访问
定义 乙 = 坐标.1  // 20

// 元组解构
定义 (x, y) = 坐标
```

**集合 (Set)**：元素不可重复，支持数学集合运算。当内容非空时可用 `{值}` 语法；空集合必须用 `集合()` 创建（因为 `{}` 被保留为空字典）。

```
定义 集合甲 = {1, 2, 3, 4}
定义 集合乙 = {3, 4, 5, 6}
定义 空集 = 集合()    // 空集合，不能用 {} （那是空字典）
定义 空字典 = {}       // 空字典

打印(交集(集合甲, 集合乙))  // {3, 4}
打印(并集(集合甲, 集合乙))  // {1, 2, 3, 4, 5, 6}
打印(差集(集合甲, 集合乙))  // {1, 2}
```

**枚举 (Enum)**：定义一组命名常量，可携带关联数据（类似 Rust 的枚举）。

```
// 简单枚举
枚举 方向
    东, 南, 西, 北

定义 朝向 = 方向.东

// 带关联数据的枚举（代数数据类型）
枚举 形状
    圆形(半径)
    矩形(宽, 高)
    三角形(底, 高)

定义 我的形状 = 形状.圆形(5.0)

// 配合模式匹配使用
函数 计算面积(图形)
    匹配 图形
        情况 形状.圆形(r): 返回 3.14159 * r * r
        情况 形状.矩形(w, h): 返回 w * h
        情况 形状.三角形(b, h): 返回 0.5 * b * h
```

### 2.3.3 字符串插值

除了使用 `+` 拼接字符串外，"道"提供更强大的字符串模板语法，使用反引号 `` ` `` 包围，用 `{}` 嵌入表达式：

```
定义 名字 = "张三"
定义 年龄 = 25

// 字符串插值
打印(`{名字} 今年 {年龄} 岁`)
// 输出：张三 今年 25 岁

// 插值中可以使用任意表达式
打印(`明年 {名字} 将 {年龄 + 1} 岁`)
// 输出：明年 张三 将 26 岁

// 多行字符串（同样支持插值）
定义 信息 = `
    姓名：{名字}
    年龄：{年龄}
    状态：{"成年" 如果 年龄 >= 18 否则 "未成年"}
`
```

### 2.3.4 解构赋值

解构赋值允许从复杂数据结构中方便地提取值，减少临时变量的使用。

```
// 列表解构
定义 [甲, 乙, 丙] = [10, 20, 30]
打印(乙)  // 20

// 使用 ... 收集剩余元素
定义 [头, ...尾] = [1, 2, 3, 4, 5]
打印(头)  // 1
打印(尾)  // [2, 3, 4, 5]

// 字典解构
定义 人 = {"姓名": "李四", "年龄": 30, "城市": "北京"}
定义 {姓名, 年龄} = 人
打印(姓名)  // "李四"
打印(年龄)  // 30

// 嵌套解构
定义 数据 = {"用户": {"名字": "王五", "地址": {"城市": "上海"}}}
定义 {用户: {名字, 地址: {城市}}} = 数据
打印(`{名字} 住在 {城市}`)  // "王五 住在 上海"

// 函数参数解构
函数 打印坐标({x, y})
    打印(`坐标：({x}, {y})`)

打印坐标({"x": 10, "y": 20})  // 坐标：(10, 20)
```

### 2.3.5 控制流

* **条件判断**：使用 `如果`-`否则如果`-`否则` 结构。代码块由缩进定义。

```
定义 分数 = 85

如果 分数 >= 90
    打印("优秀")
否则如果 分数 >= 80
    打印("良好")
否则
    打印("及格")
```

* **条件表达式**：单行条件赋值（类似三元表达式）。

```
定义 等级 = "成年" 如果 年龄 >= 18 否则 "未成年"
```

* **循环结构**：提供多种循环方式以适应不同场景。

**当循环 (While Loop):**
```
定义 计数 = 0
当 计数 < 5
    打印(计数)
    计数 = 计数 + 1
```

**遍历循环 (For Loop):**
```
定义 水果列表 = ["苹果", "香蕉", "橙子"]
遍历 每个水果 在 水果列表
    打印("我喜欢吃" + 每个水果)
```

**范围循环 (Range Loop):**
```
遍历 数字 从 1 到 5
    打印(数字) // 输出 1, 2, 3, 4, 5

// 带步长
遍历 数字 从 0 到 10 步长 2
    打印(数字) // 输出 0, 2, 4, 6, 8, 10
```

**遍历字典：**
```
定义 成绩表 = {"数学": 95, "语文": 88, "英语": 92}
遍历 科目, 分数 在 成绩表
    打印(`{科目}: {分数}分`)
```

* **循环控制**：使用 `跳出` (break) 和 `继续` (continue) 控制循环流程。

```
遍历 数字 从 1 到 100
    如果 数字 % 2 == 0
        继续  // 跳过偶数
    如果 数字 > 10
        跳出  // 超过10就停止
    打印(数字) // 输出 1, 3, 5, 7, 9
```

### 2.3.6 函数定义

使用 `函数` 关键字定义函数。函数是"道"中的一等公民，可以作为参数传递，也可以作为返回值。

```
// 定义一个普通函数
函数 加法(甲, 乙)
    返回 甲 + 乙

定义 结果 = 加法(3, 4) // 结果为 7

// 函数可以赋值给变量
定义 我的函数 = 加法
定义 结果二 = 我的函数(5, 6) // 结果二为 11
```

**默认参数与命名参数：**

```
函数 打招呼(名字, 称呼="先生", 语气="普通")
    匹配 语气
        情况 "热情": 打印(`{名字}{称呼}，好久不见！！`)
        情况 "正式": 打印(`尊敬的{名字}{称呼}，您好。`)
        情况 _: 打印(`{名字}{称呼}，你好`)

打招呼("张三")                       // 张三先生，你好
打招呼("李四", 称呼="女士")          // 李四女士，你好
打招呼("王总", 称呼="", 语气="正式") // 尊敬的王总，您好。
```

**多返回值：**

```
函数 求商和余数(被除数, 除数)
    返回 (被除数 / 除数, 被除数 % 除数)

定义 (商, 余) = 求商和余数(17, 5)
打印(`商={商}, 余数={余}`) // 商=3, 余数=2
```

**可变参数：**

```
函数 求和(...数字们)
    定义 结果 = 0
    遍历 n 在 数字们
        结果 = 结果 + n
    返回 结果

打印(求和(1, 2, 3, 4, 5)) // 15
```

## 2.4 错误处理

"道"采用结构化的异常处理机制，使用 `尝试`-`捕获`-`最终` 三段式结构，提供统一的跨范式错误处理能力。

### 2.4.1 异常捕获

```
尝试
    定义 结果 = 10 / 0
捕获 错误
    打印(`发生错误：{错误.信息}`)
最终
    打印("无论是否出错，这里都会执行")
```

### 2.4.2 抛出异常

使用 `抛出` 关键字主动抛出异常。

```
函数 取款(余额, 金额)
    如果 金额 > 余额
        抛出 错误("余额不足，当前余额：{余额}")
    返回 余额 - 金额
```

### 2.4.3 自定义异常类型

```
类型 业务异常 继承自 错误
    初始化(信息, 错误码)
        父对象.初始化(信息)
        本对象.错误码 = 错误码

尝试
    抛出 业务异常("用户不存在", 404)
捕获 异常: 业务异常
    打印(`业务错误 [{异常.错误码}]: {异常.信息}`)
捕获 异常: 错误
    打印(`系统错误: {异常.信息}`)
```

### 2.4.4 断言

用于开发阶段的快速检查。

```
函数 计算折扣(价格, 折扣率)
    断言 价格 > 0, "价格必须为正数"
    断言 0 <= 折扣率 <= 1, "折扣率必须在0到1之间"
    返回 价格 * (1 - 折扣率)
```

## 2.5 代码块与作用域

为追求语法的极致简洁，我们采用**强制缩进**来定义代码块的层级关系，类似于 Python。这种方式避免了像 `{}` 或 `开始`/`结束` 等显式块界定符，使代码在视觉上更加清晰，并强制开发者保持良好的代码风格。

**缩进规则：**
- 每级缩进使用 4 个空格（推荐）或 1 个制表符
- 同一文件中缩进风格必须一致
- 空行不影响缩进层级

**变量作用域**采用词法作用域（Lexical Scoping），即变量的可见性由其在代码中的物理位置决定。内层代码块可以访问外层变量，但不能反过来。

```
定义 全局变量 = "我是全局的"

函数 外层函数()
    定义 外层变量 = "我是外层的"
    
    函数 内层函数()
        定义 内层变量 = "我是内层的"
        打印(外层变量)  // 可以访问 ✓
        打印(全局变量)  // 可以访问 ✓
    
    内层函数()
    // 打印(内层变量) // 错误：无法访问 ✗
```

## 2.6 运算符

### 2.6.1 算术运算符

| 运算符 | 说明 | 示例 |
|--------|------|------|
| `+` | 加 | `3 + 4` → `7` |
| `-` | 减 | `10 - 3` → `7` |
| `*` | 乘 | `3 * 4` → `12` |
| `/` | 除 | `10 / 3` → `3.333...` |
| `//` | 整除 | `10 // 3` → `3` |
| `%` | 取余 | `10 % 3` → `1` |
| `**` | 幂 | `2 ** 10` → `1024` |

> 注意：`//` 作为整除运算符时，仅在表达式上下文中生效；行首的 `//` 仍然是注释。

### 2.6.2 比较运算符

| 运算符 | 说明 |
|--------|------|
| `==` | 等于 |
| `!=` | 不等于 |
| `>`, `<` | 大于、小于 |
| `>=`, `<=` | 大于等于、小于等于 |

支持链式比较（类似 Python）：

```
定义 x = 5
如果 1 < x <= 10
    打印("x 在 1 到 10 之间")
```

### 2.6.3 逻辑运算符

使用中文关键字，更具可读性：

| 运算符 | 说明 | 等价 |
|--------|------|------|
| `并且` | 逻辑与 | `&&` |
| `或者` | 逻辑或 | `\|\|` |
| `不是` | 逻辑非 | `!` |

```
如果 年龄 >= 18 并且 有驾照 == 真
    打印("可以开车")

如果 不是 已登录 或者 已过期
    打印("请重新登录")
```

### 2.6.4 成员运算符

| 运算符 | 说明 |
|--------|------|
| `在` | 是否为成员 |
| `不在` | 是否不为成员 |

```
如果 "苹果" 在 水果列表
    打印("有苹果")

如果 "键名" 不在 字典
    打印("键不存在")
```
