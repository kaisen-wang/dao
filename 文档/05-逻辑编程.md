# 5. 逻辑/规则式编程 (Logic/Rule-based Programming)

逻辑编程范式（以 Prolog 为代表）允许开发者通过声明事实和规则来描述问题，由语言引擎自动进行推理和搜索答案。将其嵌入到一种通用语言中是一个巨大的挑战，但能带来无与伦比的声明式查询能力。

## 5.1 核心机制

### 5.1.1 统一化 (Unification)

统一化是逻辑编程的核心操作，它试图通过为变量寻找合适的值来使两个逻辑项相等。例如：

- 统一 `家庭(父亲, "小明")` 和 `家庭("张三", 儿子)`
- 结果：`父亲 = "张三"` 且 `儿子 = "小明"`

### 5.1.2 回溯 (Backtracking)

当一个规则的匹配失败或用户需要所有可能的解时，系统会自动"回溯"到上一个决策点，并尝试其他的可能性。

### 5.1.3 浅层嵌入策略

我们采用"浅层嵌入" (Shallow Embedding) 的策略，在"道"的语法框架内提供逻辑编程的构造，而不是创造一个完全独立的子语言。逻辑代码块内部使用逻辑编程语义，但可以与外部的命令式/函数式代码自由交互。

## 5.2 基础语法

引入 `逻辑` 代码块，用于定义事实和规则。使用 `查询` 关键字来执行逻辑推理。**逻辑变量以问号 `?` 开头**，以区别于普通变量。

```
// 定义一个逻辑知识库
逻辑 家庭关系
    // 定义事实
    事实: 父母("张三", "小明")
    事实: 父母("李四", "小明")
    事实: 父母("张三", "小红")
    事实: 父母("小明", "乐乐")

    // 定义规则
    规则: 祖父母(?祖, ?孙) 如果
        父母(?祖, ?父) 并且 父母(?父, ?孙)

    规则: 兄弟姐妹(?甲, ?乙) 如果
        父母(?父, ?甲) 并且 父母(?父, ?乙) 并且 ?甲 != ?乙

// 在普通代码中进行查询
定义 结果 = 查询(家庭关系, 祖父母("张三", ?孙))

// 查询会返回一个包含所有可能解的列表
遍历 每个解 在 结果
    打印(`张三的孙辈是: {每个解.孙}`)
// 输出：张三的孙辈是: 乐乐

// 查询兄弟姐妹
定义 兄妹 = 查询(家庭关系, 兄弟姐妹("小明", ?谁))
遍历 每个解 在 兄妹
    打印(`小明的兄弟姐妹: {每个解.谁}`)
// 输出：小明的兄弟姐妹: 小红
```

## 5.3 否定 (Negation as Failure)

使用 `非` 关键字表示"不能被证明为真"的否定。这是 Prolog 中 `\+` 的中文等价物。

```
逻辑 权限系统
    事实: 角色("张三", "管理员")
    事实: 角色("李四", "普通用户")
    事实: 角色("王五", "普通用户")
    
    事实: 已封禁("王五")

    // 规则：有权限 = 是管理员，或者是普通用户且未被封禁
    规则: 有权限(?用户) 如果
        角色(?用户, "管理员")

    规则: 有权限(?用户) 如果
        角色(?用户, "普通用户") 并且 非 已封禁(?用户)

定义 结果 = 查询(权限系统, 有权限(?谁))
遍历 解 在 结果
    打印(`有权限的用户: {解.谁}`)
// 输出：
// 有权限的用户: 张三
// 有权限的用户: 李四
// （王五被封禁，所以不在结果中）
```

## 5.4 约束逻辑编程 (Constraint Logic Programming)

在逻辑编程基础上加入数值约束，可以解决更复杂的问题，如调度、规划、数独等。

```
逻辑 数学谜题
    // 约束：找出满足条件的数字
    规则: 谜题(?x, ?y) 如果
        ?x 在范围 1..10 并且
        ?y 在范围 1..10 并且
        ?x + ?y == 15 并且
        ?x * ?y > 50

定义 解集 = 查询(数学谜题, 谜题(?x, ?y))
遍历 解 在 解集
    打印(`x={解.x}, y={解.y}`)
// 输出所有满足 x+y=15 且 x*y>50 的整数对
```

## 5.5 剪枝 (Cut)

使用 `剪枝` 关键字阻止回溯，用于优化查询性能或控制推理行为。

```
逻辑 分类器
    // 一旦匹配到第一个规则就停止搜索
    规则: 分类(?分数, "优秀") 如果 ?分数 >= 90, 剪枝
    规则: 分类(?分数, "良好") 如果 ?分数 >= 80, 剪枝
    规则: 分类(?分数, "及格") 如果 ?分数 >= 60, 剪枝
    规则: 分类(?分数, "不及格") 如果 ?分数 >= 0
```

## 5.6 动态事实

逻辑知识库可以在运行时动态增删事实，使其能够响应程序状态的变化。

```
逻辑 用户数据库
    事实: 用户("张三", "管理员")
    事实: 用户("李四", "编辑")

// 动态添加事实
添加事实(用户数据库, 用户("王五", "读者"))

// 动态删除事实
删除事实(用户数据库, 用户("李四", "编辑"))

// 查询当前状态
定义 所有用户 = 查询(用户数据库, 用户(?名字, ?角色))
遍历 解 在 所有用户
    打印(`{解.名字}: {解.角色}`)
// 输出：
// 张三: 管理员
// 王五: 读者
```

## 5.7 逻辑编程与其他范式的交互

逻辑编程的查询结果可以无缝地传递给函数式或面向对象代码：

```
// 逻辑查询 → 函数式处理 → OOP操作
定义 推荐用户 = 查询(权限系统, 有权限(?用户))
    |> 映射(函数(解) => 解.用户)
    |> 筛选(函数(名字) => 名字 != 当前用户名)
    |> 映射(函数(名字) => 用户对象.从名字创建(名字))
```
