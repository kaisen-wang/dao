# 道语言 — 开发计划

> 本文档是"道"编程语言从设计到实现的详细开发路线图。  
> 基于 Python 3.10+ 构建第一阶段原型，后续迁移到 GraalVM/Truffle。
>
> **最新更新**: 2025-01 - 完成 Phase 2 OOP 高级特性（抽象类型、运算符重载、属性访问器）和流操作功能

---

## 📊 总体规划

| 阶段 | 时间 | 目标 | 技术栈 |
|------|------|------|--------|
| **Phase 1** | 第 1-8 周 | 核心原型：Lexer + Parser + 解释器 + REPL | Python 3.10+ |
| **Phase 2** | 第 9-20 周 | OOP + FP：类型系统、高阶函数、模式匹配、模块 | Python |
| **Phase 3** | 第 21-40 周 | 高级特性：逻辑引擎、宏系统、并发 | Python → Java |
| **Phase 4** | 第 41 周+ | 字节码编译 + GraalVM 迁移 + 生态建设 | Java/GraalVM |

---

### 3.3.3 关键设计决策

1. **展开时机**：在语法分析后，解释执行前
2. **卫生宏策略**：采用名字重写（name rewriting）
3. **AST表示**：使用Python dict/list嵌套结构
4. **执行模型**：宏是纯函数，无副作用

### 3.3.4 技术挑战与解决方案

| 挑战 | 解决方案 |
|------|----------|
| 宏展开顺序 | 后序遍历AST，先展开子节点 |
| 卫生性保证 | 为宏内变量添加唯一前缀 |
| 宏间相互调用 | 多遍展开，直到fixpoint |
| 宏展开错误 | 提供源代码位置信息 |
