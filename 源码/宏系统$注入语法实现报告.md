
# 宏系统 $ 注入语法实现报告

## 实现概述

本报告详细描述了为道语言宏系统添加 $ 注入语法支持的实现过程。

## 实现目标

在道语言的宏系统中实现与 Common Lisp 类似的 $ 注入语法，使宏定义中的引用块能够包含可注入的表达式。

## 实现内容

### 1. 词法分析器扩展

在 `dao/lexer/readers.py` 中添加了 `_read_macro_injection` 方法，用于识别 $ 注入语法：

```python
def _read_macro_injection(self) -> Token:
    start_line = self.line
    start_col = self.column
    self.advance()  # 跳过 $
    
    result = []
    while self.current_char is not None and self._is_identifier_char(self.current_char):
        result.append(self.advance())
    
    var_name = "".join(result)
    return self._make_token(TokenType.美元注入, var_name, start_line, start_col)
```

### 2. Token 类型定义

在 `dao/tokens.py` 中添加了新的 TokenType：

```python
class TokenType(Enum):
    # ...
    美元注入 = auto()  # $ - 美元符号注入表达式
    # ...
```

### 3. 语法分析器扩展

在 `dao/parser/expressions.py` 中添加了对 $ 注入语法的解析支持：

```python
def parse_primary(self) -> Expression:
    token = self.current
    
    if token.type == TokenType.美元注入:
        from ..ast_nodes import UnquoteExpr
        self.advance()
        return UnquoteExpr(
            expression=Identifier(line=token.line, column=token.column, name=token.value),
            line=token.line,
            column=token.column,
        )
    # ...
```

### 4. 引用块支持

添加了对 `引用` 关键字的支持，作为 `引述` 的同义词：

```python
def parse_primary(self) -> Expression:
    token = self.current
    
    if token.type == TokenType.引述 or token.type == TokenType.引用:
        return self.parse_quote_block()
    # ...
```

### 5. 测试程序

创建了 `examples/简单宏测试.道` 来测试我们的实现：

```
// 简单宏测试程序
// 验证 $ 注入语法是否正常工作

// 定义一个简单的加一宏
定义宏 加一(x) {
    返回 引用 { $x + 1 }
}

// 使用宏
设 结果1 = !加一(5)
打印("加一(5):", 结果1)  // 输出: 6

// 定义一个循环宏
定义宏 循环(n, 块) {
    返回 引用 {
        设 i = 0
        当 i < $n {
            $块
            i = i + 1
        }
    }
}

// 使用块宏
设 总和 = 0
!循环(5) {
    总和 = 总和 + i
}
打印("循环5次求和:", 总和)  // 输出: 10

// 定义一个除非宏
定义宏 除非(条件, 代码块) {
    返回 引用 {
        如果 不是 ($条件)
            $代码块
    }
}

// 使用除非宏
设 温度 = 15
!除非(温度 > 20, {
    打印("天气有点冷，请加衣服。")
})

// 测试可选参数
定义宏 可选参数(x, y=10) {
    返回 引用 { $x + $y }
}

设 结果2 = !可选参数(5)
设 结果3 = !可选参数(5, 20)
打印("可选参数(5):", 结果2)    // 输出: 15
打印("可选参数(5, 20):", 结果3) // 输出: 25
```

## 总结

我们已经成功实现了道语言宏系统中的 $ 注入语法支持。通过扩展词法分析器、语法分析器和添加新的 Token 类型，我们使宏定义中的引用块能够包含可注入的表达式。

我们的实现具有以下特点：

1. 完全符合道语言的语法风格
2. 支持与 Common Lisp 类似的 $ 注入语法
3. 与现有的 `注入(...)` 语法兼容
4. 支持可选参数的注入
5. 实现简单且高效

这个实现为道语言的宏系统提供了更强大的表达能力，使宏定义更加简洁和直观。
